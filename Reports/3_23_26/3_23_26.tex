\subsection{Python Terminal Script for Filtering}
\hrule
\vspace{.5cm}
Due to the fact that we to filter multiple simulations with very similar methods, I decided to make an actual script instead of just using a notebook repeatably. The largest upside is that this can easily be run on a compute node, which makes filtering larger sims easier. Also helps make sure any results are reproducable.

Takes a user input via the CLI (see below), filters for all \verb|BH_Sol| systems, and returns the population file with said systems. Additionally, it returns a .csv of just the \verb|BH_Sol| rows. While running it provides feedback via the terminal stating how many \verb|BH_MS| systems it found, as well as how many are the expected S1:S2 configuration. If any are S2:S1, it prints a warning as well as the total count.\\

Need to further refine it, but there's also a default SBATCH script to help expidite the submission process.

\textbf{Usage Example}
\begin{minted}{bash}
  python BH-Sol_Filter.py /Data/1e+00_Zsun_population.h5 -o ./testResults
\end{minted}

\textbf{Arguments}
\begin{center}
\begin{tabular}{|c|c|c|}
\hline
 Argument & Use & Ex value \\
 \hline
 \verb|-o| & output location  & .results\\
 \verb|-outputName| & name for the outputed files\footnote{If a name is not specificed it will use the orignal filename, appending a specifier to the front}  & \verb|BH_Sol_Systems|\\
 \verb|-overwrite| & Whether to overwrite existing results, T/F bool. Defaults to F& True\\
 \verb|--maxMass| & Max mass used when filtering, defaults to 3\(M_\odot\)& 3.5\\
 \verb|--maxPeriod| & Max period used when filtering, defaults to 3.5 days & 4\\
\hline
\end{tabular}
\end{center}

Can be found at \url{https://github.com/PiersonLip/stellarBHs/blob/main/FilteringScripts/BH-Sol_Filter.py}

\subsection{Pop 1}
\hrule
\vspace{.5cm}

When filtering via notebook and script, both returned 6 systems that had \(p_{orb} < 3.5, M_{\odot,2} < 3, t < 10e9\)

These systems all evolved via the same channel, \verb|ZAMS_oCE1_CC1_oRLO2_CC2|. Via inspection of the history df and the oneline, it is evident that the systems evolve detached, then into post-MS \textbf{unstable} RLO via S1 \(\rightarrow\) S2. This period decrease leads to common envolope, which then \textit{greatly} decreases the orbtial period. The envolope is then ejected, leaving a depleted stripped He core\footnote{I need to read a lot more about CE evo, but my intutuion feels weird about the fact that just the envolope of \textit{one} of the stars is ejected}. The core then collapses in a couple of thousand years, leaving the system in a BH-MS state with low \(p_{orb}\)/seperation. 

These systems then evolve in a semi-peculiar way, where it is highy likely that there will be RLO from S2 onto S1, and because \(S1 > S2\), the transfer is stable, leading to \textit{very} prolonged stable RLO (seemingly on the scale of  \(> 10 Gyr\)), which leaves S2 at low masses (\(< .08 M_\odot\)).

Interestingly, the inital period for all the systems is close to \(~4000 d\), with mean of 4022 and a std of only 151. However, a larger sample size is needed to confirm this.

Additionally, the inital masses of S1 and S2 appear to be quite constrained as well.

\subsection{Full \(P_{orb}\)}
\hrule
\vspace{.5cm}
When filtering population 2 with a max \(p_{orb} = 300 \text{days}\) we find 46 BH-Sol systems, all of these (except one!) evolve via the same CE channel. This suggests that there isn't a unique idenitiy placed on BH-Sol systems with \(p_{orb} \leq 3 \text{days}\)\footnote{This orbtial period distribution is possibly due to natal kick velocities, and is suggested by the literature, but I could not find a strong link in the limited data.}, and thus it makes more sense to investigate the population as a whole. Note that there is one system which does \textit{not} do RLO before CC1, and only reaches a solar-like mass after RLO2 (upon which it also fails), so I'm excluding it for now. Likely important to keep in mind though.
